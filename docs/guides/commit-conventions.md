# Правила оформления коммитов

Проект использует [Conventional Commits](https://www.conventionalcommits.org/ru/) для автоматической генерации CHANGELOG и семантического версионирования.

## Формат коммита

```
<тип>(<область>): <описание>

[тело]

[футер]
```

### Язык

- **Префикс** (`feat`, `fix`, `refactor`, ...) и **scope** — всегда на **английском**
- **Описание** (subject) — на **русском языке** с естественными англицизмами
- Технические термины пишутся **английскими буквами** как есть: landing, hero, i18n, SSR, hook, store, layout и т.д.

### Примеры

```
feat(landing): добавить hero секцию с CTA
fix(i18n): исправить определение языка из URL
refactor(shared): вынести analytics в отдельный модуль
docs: обновить README
chore: обновить зависимости
```

## Типы коммитов

| Тип        | Описание                                 | Попадает в CHANGELOG |
| ---------- | ---------------------------------------- | -------------------- |
| `feat`     | Новая функциональность                   | Да                   |
| `fix`      | Исправление бага                         | Да                   |
| `docs`     | Изменения в документации                 | Нет                  |
| `style`    | Форматирование, пробелы, точки с запятой | Нет                  |
| `refactor` | Рефакторинг без изменения поведения      | Нет                  |
| `perf`     | Улучшение производительности             | Да                   |
| `test`     | Добавление или исправление тестов        | Нет                  |
| `build`    | Изменения в сборке или зависимостях      | Нет                  |
| `ci`       | Изменения в CI/CD конфигурации           | Нет                  |
| `chore`    | Прочее (не затрагивает src/)             | Нет                  |
| `revert`   | Откат предыдущего коммита                | Да                   |

## Версионирование

Проект использует [Semantic Versioning](https://semver.org/):

```bash
pnpm release:dry      # предпросмотр
pnpm release:patch    # 0.1.0 → 0.1.1
pnpm release:minor    # 0.1.0 → 0.2.0
pnpm release:major    # 0.1.0 → 1.0.0
```

## Валидация

Коммиты автоматически проверяются через `commitlint` (husky hook `commit-msg`).
